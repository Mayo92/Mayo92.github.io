Java虚拟机结构解析

JVM整体结构讲解

编译流程

类加载器

Bootstrap、Extension加载JDK特定的jar包

App应用程序所用到的加载器

Custom可以自定义的加载器。可以重写classloader，让这个classloader加载自定义的class。可以实现动态加载的理论依据与技术手段

类加载器将class字节码加载到JVM流程

  • Loading：类的信息从文件中获取并且载入到JVM的内存里
  • Verifying：检查读入的结构是否符合JVM规范的描述
  • Preparing：分配一个结构用来存储类信息
  • Resolving：把这个类的常量池中的所有的符号引用改变成直接引用
  • Initializing：执行静态初始化程序，把静态变量初始化成指定的值

内存管理

Java栈区

  • 作用：它存放的是java方法执行时的所有的数据（描述java方法执行的一个完整的内存模型）
  • 组成：由栈帧组成，一个栈帧代表一个方法的执行

java栈帧

  • 每个方法从调用到执行完成就对应一个栈帧在虚拟机中入栈到出栈

A方法调用到了B方法，java虚拟机创建一个保存B方法的栈帧压入到栈区中，B方法执行完成后返回到A方法中时，这个栈帧弹出

  • 每个栈帧包含：局部变量表、栈操作数、动态链接、方法出口

本地方法栈

  • 作用：本地方法栈是专门为Native方法服务的

方法区

  • 存储被虚拟机加载的类信息、常量、静态变量、即使编译器编译后的数据（永远占据内存）

堆区

  • 作用：所有通过new创建的对象的内存都在堆中分配
  • 特点：是虚拟机中最大的一块内存，是GC要回收的部分

可以让开发人员动态调整Young与Old的大小

垃圾回收

垃圾收集算法

  • 引用计数算法（1.2之前）

A与B的引用计数都是1，但是这两个对象都是不可达的，但引用计数算法下不能被回收

  • 可达性算法（根搜索性算法）

引用的类型

  • 强引用、软引用、弱引用、虚引用
  • 最常使用的就是强引用、弱引用

弱引用的创建

Object obj = new Object();//强引用
//Object对象有两个引用指向它obj、wf
WeakReference<Object> wf = new WeakReference<Object>(obj);
obj = null;
wf.get();//使用的时候要判断是否为空

垃圾回收算法

  • 标记-清除算法

好处：不需要进过对象的移动并且仅对不存活的对象进行处理，在存活对象比较多的情况下很高效。坏处：会造成内存碎片，不利于后续对内存的分配

  • 复制算法

优点：存活对象少时候高效。缺点：需要一块内存作为交换空间来进行对象的移动

  • 标记-整理算法

在标记-清除算法基础上，成本更高但解决了内存碎片问题
这三种算法在虚拟机中是结合使用的。

触发回收

  • Java虚拟机无法再为新的对象分配内存空间了
  • 手动调用System.gc()方法（不推荐）
  • 低优先级的GC线程，被运行时就会执行GC

Dalvik与JVM的不同

  • 执行的文件不同，一个是class，一个是dex
  • 类加载的系统与JVM区别较大
  • 可以同时存在多个DVM（某一个应用程序的虚拟机挂掉不会影响其他，保证系统稳定性）
  • Dalvik是基于寄存器的（运行的更快），JVM是基于栈的（方法调用在栈中完成）

ART比Dalvik有哪些优势

  • DVM使用JIT（每次运行都会将字节码转换成本地机器码再执行）来将字节码转换成机器码，效率低
  • ART采用了AOT（安装的时候将字节码转换成本地机器码）预编译技术，执行速度更快
  • ART会占用更多的应用安装时间和存储空间（以空间换时间）

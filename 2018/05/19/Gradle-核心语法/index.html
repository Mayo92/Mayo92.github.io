Gradle-核心语法

2018-05-19
#Gradle语法

第一节的文件类型

基础语法讲解

变量详解

变量的类型

int x = 10
println x.class

double y = 3.14
println y.class

Groovy中没有基本类型变量，所有类型都是对象类型

变量的定义

强类型定义方法

像Java一样，直接定义int类型、double类型等，声明这个变量的时候把类型确定了

弱类型def定义方法
def x1 = 11
println x1.class

def y1 = 3.1415
println y1.class

def name = 'ss'
println name.class

根据后面的值就可以推断出变量的类型，编译器会自动推断

两种方式使用场景

如果这个变量只用于自己而不会使用其他类或其他应用模块，推荐定义成def。如果这个变量要用于其他模块或者其他类，推荐使用强类型定义方式，因为不能动态的转换成其他类型，保证外接传给我们的值一定是正确的，反之调用方会疑惑

def x1 = 11
x1 = 'test'
println x1.class

def声明的变量可以随时改变类型，相当于Object类型

字符串详解

常用的三种定义方式

单引号定义

内容就是字符串的内容，且不能更改。特殊字符跟Java一样，通过\转义

def name = 'a single string'
print name.class

三引号定义
def thupleName = '''three single string'''
println thupleName.class

可以直接指定格式，有格式的建议使用三引号

def thupleName2 = '''line one
line two
line three
'''
println thupleName2

可以添加一个反斜杠，让格式更统一

def thupleName2 = '''\
line one
line two
line three
'''
双引号定义

最常用，可扩展字符串

def doubleName = "common string"
println doubleName.class


def name2 = "zz"
def hello = "hello:${name2}"
println hello
println hello.class

可以使用${}扩展，并且如果有扩展表达式就改变了类型，{}可以是任意的表达式

def sum = "sum of 2 and 3 equals ${2 + 3}"
println sum

两种类型可以互相转换

String与GString可以相互传递，相互调用

def sum = "sum of 2 and 3 equals ${2 + 3}"

def result = echo(sum)
println result
println result.class

String echo(String message){
    return message
}

闭包

groovy中闭包基础详解

闭包定义及调用

闭包就是一段代码块

def clouser = { println 'hello groovy!' }
//执行
clouser.call()
//第二种调用
clouser()

闭包参数

->之前为参数部分，->之后为闭包体

def clouser = {String name  -> println "hello ${name}" }
clouser.call('groovy')
clouser('groovy')

多个参数

def clouser = {String name,int age  -> println "hello ${name},My age is ${age}" }
def name = 'groovy'
clouser(name,4)

默认参数it,所有闭包都有的默认参数，不需要显示申明

def clouser = {println "hello ${it}" }
def name = 'groovy'
clouser(name)

闭包返回值

闭包一定会有返回值，下面代码return改为println后，返回值为null

//def clouser = {String name  -> return   "hello ${name}" }
def clouser = {String name  -> println  "hello ${name}" }
def result = clouser('groovy')
println result

groovy中闭包使用详解

与基本类型的结合使用

可以在DefaultGroovyMethods方法中查看实现

闭包作为方法参数的最后一个可以不写在括号里

阶乘方法

int x = fab(5)
println x
int y = fab2(5)
println y

//求指定number的阶乘
int fab(int number) {
    int result = 1
    1.upto(number , { num -> result *= num})
    return result
}

int fab2 (int number){
    int result = 1
    number.downto(1) {num -> result*= num}
    return result
}

累计求和方法

//times方法起始值始终是0
int call (int number) {
    int result = 0
    number.times {
        num -> result+=num
    }
    return result
}
//0加到4
int z = call(5)
println z

与String结合使用

each遍历，each的返回值还是str的值（调用者本身）

String str = 'the 2 and 3 is 5'
str.each {print it}

find方法，查找符合条件的第一个,闭包是一个Boolean类型返回值

println str.find {it.isNumber()}

findAll方法,遍历字符串找到所有符合的放到集合里，闭包是一个Boolean类型返回值

def list = str.findAll {it.isNumber()}
println list.toListString()

any方法，只要满足条件就返回true，闭包是一个Boolean类型返回值

def result = str.any { it.isNumber() }
println result

every方法，每一项都满足返回true，闭包是一个Boolean类型返回值

println str.every {it.isNumber()}

collect方法，将字符串的每一个元素都应用闭包，会产生一个新的结果，将结果添加到list返回

def list2 = str.collect {it.toUpperCase()}
println list2.toListString()

groovy中闭包进阶详解

三个重要的变量

def scriptClouser = {
    println "scriptClouser this:"+this
    println "scriptClouser owner:"+owner
    println "scriptClouser delegate:"+delegate
}
scriptClouser.call()

  • this：与java一样，代表闭包定义处的类，如上例指向脚本所对应的类
  • owner：代表闭包定义处的类或者对象

  • delegate：代表任意对象，有默认值与owner一致

    class Person {

    def static classClouser = {
    println "classClouser this:"+this
    println "classClouser owner:"+owner
    println "classClouser delegate:"+delegate
}

def static say() {
    def classClouser = {
        println "methodClassClouser this:"+this
        println "methodClassClouser owner:"+owner
        println "methodClassClouser delegate:"+delegate
    }

    classClouser.call()
}

    }
    Person.classClouser.call()
    Person.say()

上图指向的是类的字节码而不是对象，因为是static的

def outClouser = {
    def innerClouser = {
        println "innerClouser this:"+this
        println "innerClouser owner:"+owner
        println "innerClouser delegate:"+delegate
    }
    innerClouser.call()
}
outClouser.call()

总结：大多数情况，三者一致。如果直接在类和方法中定义闭包，三个变量是一样的。如果再闭包中又嵌套定义一个闭包this与owner、delegate的值不一样，owner、delegate会指向离他最近的包的对象

Person p = new Person()
def outClouser = {
    def innerClouser = {
        println "innerClouser this:"+this
        println "innerClouser owner:"+owner
        println "innerClouser delegate:"+delegate
    }
    innerClouser.delegate = p //修改默认的delegate
    innerClouser.call()
}
outClouser.call()

人为修改delegate所指向的对象，delegate与owner不一样。this与owner不能修改

闭包委托策略（三个关键字的作用）

class Student{
    String name
    def pretty = {
        "My name is ${name}"
    }
    String toString(){
        pretty.call()
    }
}
class Teacher{
    String name
}
def stu = new Student(name: 'krido')
def tea = new Teacher(name: 'mayo')
stu.pretty.delegate = tea
//更改委托策略。
stu.pretty.resolveStrategy = Closure.DELEGATE_FIRST
println stu.toString()


class Student{
    String name
    def pretty = {
        "My name is ${name}"
    }
    String toString(){
        pretty.call()
    }
}
class Teacher{
    String name1
}
def stu = new Student(name: 'krido')
def tea = new Teacher(name1: 'mayo')
stu.pretty.delegate = tea
//更改委托策略
stu.pretty.resolveStrategy = Closure.DELEGATE_FIRST
println stu.toString()

因为tea中没有name，Closure.DELEGATE_FIRST找不到会去OWNER中寻找。如果上例改成DELEGATE_ONLY，会报错，因为没有找到name属性

委托策略默认是OWNER_FIRST，表明闭包中的变量、方法等都是首先从OWNER对象所指向的对象处去寻找。一共有四组（OWNER_FIRST、DELEGATE_FIRST、OWNER_ONLY、DELEGATE_ONLY）

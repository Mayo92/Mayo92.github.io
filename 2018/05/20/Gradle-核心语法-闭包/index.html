闭包

1.groovy中闭包基础详解

1.1 闭包定义及调用（与方法对比学习）

闭包就是一段代码块

def clouser = { println 'hello groovy!' }
//执行
clouser.call()
//第二种调用
clouser()

1.2 闭包参数

->之前为参数部分，->之后为闭包体

def clouser = {String name  -> println "hello ${name}" }
clouser.call('groovy')
clouser('groovy')

多个参数

def clouser = {String name,int age  -> println "hello ${name},My age is ${age}" }
def name = 'groovy'
clouser(name,4)

默认参数it,所有闭包都有的默认参数，不需要显示申明

def clouser = {println "hello ${it}" }
def name = 'groovy'
clouser(name)

1.3 闭包返回值

闭包一定会有返回值（与方法不同），下面代码return改为println后，返回值为null

//def clouser = {String name  -> return   "hello ${name}" }
def clouser = {String name  -> println  "hello ${name}" }
def result = clouser('groovy')
println result

2.groovy中闭包使用详解

2.1 与基本类型的结合使用

可以在DefaultGroovyMethods方法中查看实现，参数的个数查看源码方法里call()的个数

闭包作为方法参数的最后一个可以不写在括号里

不用指定闭包参数类型，因为groovy是动态类型的，会在运行时自己推断

阶乘方法

int x = fab(5)
println x
int y = fab2(5)
println y

//求指定number的阶乘
int fab(int number) {
    int result = 1
    1.upto(number , { num -> result *= num})
    return result
}

int fab2 (int number){
    int result = 1
    number.downto(1) {num -> result*= num}
    return result
}

累计求和方法

//times方法起始值始终是0
int call (int number) {
    int result = 0
    number.times {
        num -> result+=num
    }
    return result
}
//0加到4
int z = call(5)
println z

2.2 与String结合使用

each遍历

each的返回值还是str的值（调用者本身）

String str = 'the 2 and 3 is 5'
str.each {print it}

find方法

查找符合条件的第一个,闭包是一个Boolean类型返回值

println str.find {it.isNumber()}

findAll方法

遍历字符串找到所有符合的放到集合里，闭包是一个Boolean类型返回值

def list = str.findAll {it.isNumber()}
println list.toListString()

any方法

只要满足条件就返回true，闭包是一个Boolean类型返回值

def result = str.any { it.isNumber() }
println result

every方法

每一项都满足返回true，闭包是一个Boolean类型返回值

println str.every {it.isNumber()}

collect方法

将字符串的每一个元素都应用闭包，会产生一个新的结果，将结果添加到list返回

def list2 = str.collect {it.toUpperCase()}
println list2.toListString()

3.groovy中闭包进阶详解（了解即可）

3.1 三个重要的变量

def scriptClouser = {
    println "scriptClouser this:"+this
    println "scriptClouser owner:"+owner
    println "scriptClouser delegate:"+delegate
}
scriptClouser.call()


  • this：与java一样，代表闭包定义处的类，如上例指向脚本所对应的类
  • owner：代表闭包定义处的类或者对象（如闭包内部还可以定义闭包，此时为闭包定义处的闭包）
  • delegate：代表任意对象，有默认值与owner一致

例子

//脚本中定义了一个内部类
class Person {

    def static classClouser = {
        println "classClouser this:"+this
        println "classClouser owner:"+owner
        println "classClouser delegate:"+delegate
    }

    def static say() {
        def classClouser = {
            println "methodClassClouser this:"+this
            println "methodClassClouser owner:"+owner
            println "methodClassClouser delegate:"+delegate
        }

        classClouser.call()
    }
}
Person.classClouser.call()
Person.say()

上图指向的是类的字节码而不是对象，因为是static的

//闭包中定义闭包
def outClouser = {
    def innerClouser = {
        println "innerClouser this:"+this
        println "innerClouser owner:"+owner
        println "innerClouser delegate:"+delegate
    }
    innerClouser.call()
}
outClouser.call()

总结：大多数情况，三者一致。如果直接在类和方法中定义闭包，三个变量是一样的。如果再闭包中又嵌套定义一个闭包，this与owner、delegate的值不一样，owner、delegate会指向离他最近的闭包的对象

//修改delegate
Person p = new Person()
def outClouser = {
    def innerClouser = {
        println "innerClouser this:"+this
        println "innerClouser owner:"+owner
        println "innerClouser delegate:"+delegate
    }
    innerClouser.delegate = p //修改默认的delegate
    innerClouser.call()
}
outClouser.call()

人为修改delegate所指向的对象，delegate与owner不一样。this与owner不能修改

3.2 闭包委托策略（三个关键字的作用）

更改属性

class Student{
    String name
    def pretty = {
        "My name is ${name}"
    }
    String toString(){
        pretty.call()
    }
}
class Teacher{
    String name
}
def stu = new Student(name: 'krido')
def tea = new Teacher(name: 'mayo')
stu.pretty.delegate = tea
//更改委托策略。改为delegate后，会去找tea的name，如果没有name属性，会从owner中去找，如果此时是DELEGATE_ONLY找不到则会报错
stu.pretty.resolveStrategy = Closure.DELEGATE_FIRST
println stu.toString()


class Student{
    String name
    def pretty = {
        "My name is ${name}"
    }
    String toString(){
        pretty.call()
    }
}
class Teacher{
    String name1
}
def stu = new Student(name: 'krido')
def tea = new Teacher(name1: 'mayo')
stu.pretty.delegate = tea
//更改委托策略
stu.pretty.resolveStrategy = Closure.DELEGATE_FIRST
println stu.toString()

改成name1后，因为tea中没有name，Closure.DELEGATE_FIRST找不到会去OWNER中寻找。如果上例改成DELEGATE_ONLY，会报错，因为tea没有找到name属性

委托策略默认是OWNER_FIRST，表明闭包中的变量、方法等都是首先从OWNER对象所指向的对象处去寻找。一共有四组（OWNER_FIRST、DELEGATE_FIRST、OWNER_ONLY、DELEGATE_ONLY）

通过不同的委托策略的指定，我们可以修改闭包所要调用的变量和方法

更改方法

class Student2{
    def self = {
        mySelf()
    }

    String mySelf(){
        println("i'm student")
    }
}
class Teacher2{

    String mySelf(){
        println("i'm teacher")
    }
}
def stu2 = new Student2()
def tea2 = new Teacher2()
stu2.self.delegate = tea2
stu2.self.resolveStrategy = Closure.DELEGATE_ONLY
stu2.self()

上面会输出”i’m teacher”

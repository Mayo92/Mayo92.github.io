第一行代码笔记-Material Design

2018-03-18
#Material Design

什么是Material Design

界面设计语言，包含视觉、运动、交互效果。开发人员使用Design Support库即可。

Toolbar

官方已经不建议使用ActionBar了，推荐使用Toolbar。
Toolbar不仅继承了ActionBar的所有功能，而且灵活性很高，可以配合其他控件完成一些Material Design的效果。

为什么默认会显示ActionBar

这是根据项目的主题而来的，从AndroidManifest.xml的android:theme找到res/values/styles.xml，找到主题的parent是Theme.AppCompat.Light.DarkActionBar，这个DarkActionBar是一个深色的ActionBar主题，所以才会出现ActionBar。

指定不带ActionBar的主题，通常有Theme.AppCompat.NoActionBar和Theme.AppCompat.Light.NoActionBar这两种主题可选。其中Theme.AppCompat.NoActionBar表示深色主题，界面主体为深色，陪衬颜色为淡色。Theme.AppCompat.Light.NoActionBar表示淡色主题，界面主体为淡色，陪衬颜色为深色。

AppTheme中的属性重新，这里重写了colorPrimary、colorPrimaryDark、colorAccent这三个颜色属性，如下图：

除上述3个属性，还可以通过textColorPrimary、windowBackground和navigationBarColor等属性来控制更多的颜色。不过colorAccent这个属性比较难理解，它不只是用来指定这样一个按钮的颜色，更多的表达强调的意思，比如一些控件的选中状态也会使用colorAccent的颜色。

添加Toolbar

<android.support.v7.widget.Toolbar
    android:layout_width="match_parent"
    android:layout_heiht="?attr/actionBarSize"
    android:background="?attr/colorPrimary"
    android:theme="@style/ThemeOverlay.AppCompat.Dark.ActionBar"
    app:popupTheme="@style/ThemeOverlay.AppCompat.Light"/>

使用了xmlns:android命名空间，之后可以使用app:attribute这样的写法了。因为Material Design是Android5.0出现的，而很多Material属性在5.0之前的系统不存在，为了兼容，不能使用android: attribute，而应该使用app:attribute。

这里的高度为actionBar的高度。由于在style.xml中将程序主题指定成了淡色，因此Toolbar也是淡色，而Toolbar上面的各个元素会自动使用深色，为了和主题颜色区分。但这样很难看。这里使用android:theme，使Toolbar单独使用深色主题，但这样如果Toolbar里有菜单按钮，那么弹出来的也是深色主题，这样也难看，所以使用app:popupTheme属性单独将弹出来的菜单指定淡色主题。

添加Toolbar，在onCreate()里

Toolbar toobar = findViewById(R.id.toobar);
setSupportActionBar(toobar);

修改标题栏显示文字内容，在manifest中

<Activity
    android:label="Fruits">
</Activity>

如果没有指定，会默认使用application中指定的label内容，即应用名称。

添加action按钮，res/menu下创建toolbar.xml

<menu xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto">

    <item
        android:id="@+id/backup"
        android:icon="@drawable/ic_ backup"
        android:title="backup"
        app:showAsAction="alawys"/>
    <item
        android:id="@+id/delete"
        android:icon="@drawable/ic_ delete"
        android:title="delete"
        app:showAsAction="ifRoom"/>
    <item
        android:id="@+id/settings"
        android:icon="@drawable/ic_ settings"
        android:title="settings"
        app:showAsAction="never"/>
</menu>

app:showAsAction指定按钮显示位置，alawys永远显示，如果屏幕空间不够则不显示；ifRoom表示屏幕空间够显示，不够不显示；never永远显示在菜单中。注意，Toolbar中的action按钮只会显示图标，菜单中的action按钮只会显示文字。

滑动菜单

DrawerLayout

所谓滑动菜单就是将一些菜单选项隐藏，然后可以通过滑动的方式将菜单显示出来，这种方式即节省了屏幕空间，又实现了非常好的动画效果，是Material Design中推荐的做法。

DrawerLayout是一个布局，允许放入两个直接子控件，第一个控件是主屏幕显示的内容，第二个子控件是滑动菜单显示的内容。

<?xml version="1.0" encoding="utf-8"?>
<android.support.v4.widget.DrawerLayout 
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:id="@+id/drawerLayout"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <FrameLayout
        android:layout_width="match_parent"
        android:layout_height="match_parent">
        
        <android.support.v7.widget.Toolbar
            android:layout_width="match_parent"
            android:layout_heiht="?attr/actionBarSize"
            android:background="?attr/colorPrimary"
            android:theme="@style/ThemeOverlay.AppCompat.Dark.ActionBar"
            app:popupTheme="@style/ThemeOverlay.AppCompat.Light"/>
    </FrameLayout>
    
    <TextView
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:layout_gravity="start"/>

</android.support.v4.widget.DrawerLayout>

DrawerLayout没有限制只能使用固定的控件。第二个控件android:layout_gravity是必须的，需要告诉DrawerLayout滑动菜单是在屏幕的左边还是右边.left左边，right右边，start会根据系统语言进行判断。

向左滑动菜单或点击菜单以外的区域，都可以让菜单关闭。

在Toolbar最左边加一个导航按钮，防止用户不知道屏幕左边是可以拖动的。

在onCreate()中添加

Toolbar toobar = findViewById(R.id.toobar);
setSupportActionBar(toobar);
mDrawerLayout = findViewById(R.id.drawerLayout);
ActionBar actionBar = getSupportActionBar();
if(actionBar != null){
    actionBar.setDisplayHomeAsUpEnabled(true);
    actionBar.setHomeAsUpIndicator(R.drawable.ic_menu);
}

在onOptionsItemSeleted()中添加

case android.R.id.home:
    mDrawerLayout.openDrawer(GravityCompat.START);
    break;

getSupportActionBar()得到ActionBar实例，具体实现是由Toolbar来完成的。setDisplayHomeAsUpEnabled()方法让导航按钮显示出来，调用setHomeAsUpIndicator()方法设置一个导航按钮图标。实际上，Toolbar最左侧的按钮就叫做HomeAsUp按钮，它默认的图标是一个返回按钮，返回上一个活动。这里我们将它默认的样式和作用进行了修改。

HomeAsUp的ID永远都是android.R.id.home。调用openDrawer()方法将滑动菜单显示出来，需传入一个Gravity参数，为了与XML一致，这里传入了GravityCompat.START。

NavigationView

可以在滑动菜单也定制任意的布局，不过谷歌提供了NavigationView。NavigationView是Design Support库中提供的一个控件，不仅严格按照MD的要求设计，而且可以将滑动菜单页的实现变得非常简单。

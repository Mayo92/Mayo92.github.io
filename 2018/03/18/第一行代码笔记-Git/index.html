    忽略文件

    提供一种可配性很强的机制来允许用户将指定的文件或目录排除在版本控制之外，它会检查代码仓库的目录下是否存在一个名为.gitignore的文件，存在的话会一行一行读取这个文件的内容，并把每一行指定的文件或目录排除在版本控制之外，文件或目录可以使用”*”通配符的。

    AS在创建项目的时候会自动帮我们创建两个.gitignore文件，一个在根目录下，一个在app模块下。

    配置完成之后，就可以提交代码，先使用add命令将所有文件进行添加

    git add .

    然后执行commit命令完成提交

    git commit -m "first commit"

    查看修改内容

    查看文件修改的情况
    git status
    查看文件修改的内容
    git diff
    查看某个文件修改的内容
    git diff app/src/main/java/com/example/providertest/MainActivity.java

    撤销未提交的修改

    只要代码没有提交，所有修改的内容都是可以撤销的
    git checkout app/src/main/java/com/example/providertest/MainActivity.java
    对于已添加的文件我们应该先对其取消添加，然后才可以撤回提交，取消添加如下：
    git reset HEAD app/src/main/java/com/example/providertest/MainActivity.java

    查看提交记录

    每次提交记录都会包含提交id、提交人、提交日期、描述信息
    git log
    只想查看其中一条记录，可以在命令中指定该记录的id，并加上-1参数表示只想看到一行记录
    git log 1fa380xxxxxxx -1
    查看这条记录具体修改什么内容，在命令中加入-p参数
    git log 1fa380xxxxxxx -1 -p

    分支的作用

    在现有代码的基础上开辟一个分叉口，使得代码可以在主干线和分支线上同时进行开发，且互相不会影响。

忽略文件

提供一种可配性很强的机制来允许用户将指定的文件或目录排除在版本控制之外，它会检查代码仓库的目录下是否存在一个名为.gitignore的文件，存在的话会一行一行读取这个文件的内容，并把每一行指定的文件或目录排除在版本控制之外，文件或目录可以使用”*”通配符的。

AS在创建项目的时候会自动帮我们创建两个.gitignore文件，一个在根目录下，一个在app模块下。

配置完成之后，就可以提交代码，先使用add命令将所有文件进行添加

git add .

然后执行commit命令完成提交

git commit -m "first commit"

查看修改内容

查看文件修改的情况
git status
查看文件修改的内容
git diff
查看某个文件修改的内容
git diff app/src/main/java/com/example/providertest/MainActivity.java

撤销未提交的修改

只要代码没有提交，所有修改的内容都是可以撤销的
git checkout app/src/main/java/com/example/providertest/MainActivity.java
对于已添加的文件我们应该先对其取消添加，然后才可以撤回提交，取消添加如下：
git reset HEAD app/src/main/java/com/example/providertest/MainActivity.java

查看提交记录

每次提交记录都会包含提交id、提交人、提交日期、描述信息
git log
只想查看其中一条记录，可以在命令中指定该记录的id，并加上-1参数表示只想看到一行记录
git log 1fa380xxxxxxx -1
查看这条记录具体修改什么内容，在命令中加入-p参数
git log 1fa380xxxxxxx -1 -p

分支的作用

在现有代码的基础上开辟一个分叉口，使得代码可以在主干线和分支线上同时进行开发，且互相不会影响。

查看当前版本库中有哪些分支，前面有*说明代码在哪个分支上：

git branch

创建一个分支,名为version1.0：

git branch version1.0

切换到version1.0这个分支：

git checkout version1.0

在version1.0分支上修改并提交的代码将不会影响到master分支，同理，master分支上修改并提交的代码也不会影响到version1.0分支，最好的办法是使用merge命令来完成合并操作：

git checkout master
git merge version1.0

这两行代码就可以在version1.0分支上修改并提交的内容合并到master分支，合并后可能存在代码冲突的情况，需要自行修改

删除分支：

git branch -D version1.0

与远程版本库协作

养成经常从版本库获取最新代码的习惯

将代码下载到本地：

git clone https://github.com/example/test.git

本地修改的内容同步到远程版本库：

git push origin master

origin部分指定的是远程版本库的Git地址，master部分指定的是同步到哪一个分支，上述命令就完成了将本地代码同步到https://github.com/example/test.git这个版本库的master分支上的功能。

将远程版本库上的修改同步到本地，Git提供两种命令完成此功能，分别是fetch和pull，fetch的语法规则和push差不多，如下所示：

git fetch origin master

执行上述命令后，会将远程版本库上的代码同步到本地，同步下来的代码不会合并到任何分支上去，而是会存放到一个origin/master分支上，这时我们可以通过diff命令来查看远程版本库上到底修改了哪些东西：

git diff origin/master

再调用merge命令将origin/master分支上的修改合并到主分支上即可：

git merge origin/master

pull命令则是相当于将fetch和merge这两个命令放在一起执行了，它可以从远程版本库上获取最新的代码合并到本地：

git pull origin master

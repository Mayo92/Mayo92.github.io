第一行代码笔记-LBS

简介

基于位置的服务，Location Based Service。利用无线电通讯网络或者GPS等定位方式来确定移动设备所在的位置。

GPS定位是基于手机内置的GPS硬件直接和卫星交互来获取当前的经纬度信息，精确度非常高，缺点是只能在室外使用，室内基本无法接收到卫星信号。

网络定位根据手机当前网络附近的三个基站进行测试，计算出手机和每个基站的距离，再通过三角定位确定大概的位置，精度一般，室内外都可使用。

keytool -list -v -keystore <签名文件路径>

LBS SDK

libs目录下的内容分为两部分，BaiduLBS_Android.jar这个文件是Java层要使用到的，其他子目录下的so文件是Native层要用到的。so文件是用c/c++编写，然后再用NDK编译出来的。

app目录下有一个libs目录，用来存放所有的Jar包。展开src/main目录，创建一个jniLibs目录，用来存放so文件。

虽然所有新创建的项目中,app/build.gradle文件都会默认配置以下这段声明：

implementation fileTree(dir: 'libs', include: ['*.jar'])

这个表示会将libs目录下所有以.jar结尾的文件添加到当前项目的引用中，但由于我们是直接将Jar包复制到libs目录下的，并没有修改gradle文件，因此不会弹出我们熟悉的Sync Now提示，需手动点击。

选择定位模式

GPS定位功能必须要由用户主动去开启才行。SDK有三个模式可选，Hight_Accuracy、Battery_Saving、Device_Sensors。

  • Hight_Accuracy高精确模式，会在GPS信号正常的情况下优先使用GPS定位，无法接收GPS信号的时候使用网络定位。默认模式。
  • Battery_Saving节电模式，使用网络定位
  • Device_Sensors传感器模式，GPS定位。

让地图显示出来
移动到我的位置

SDK提供一个BaiduMap类，是地图的总控制器，调用MapView的getMap()方法就能获取BaiduMap的实例

BaiduMap baiduMap = mapView.getMap();

百度地图缩放级别的取值范围限定在3到19之间，小数点位的值也是可取的，值越大，地图显示的信息就越精细，设置成12.5：

MapStatusUpdate update = MapStatusUpdateFactory.zoomTo(12.5f);
baiduMap.animateMapStatus(update);

LatLng用于存放经纬度值，构造方法接收两个参数，第一个是纬度值，第二个是经度值。调用MapStatusUpdateFactory的newLatLng()方法将LatLng对象传入，newLatLng()方法返回的也是一个MapStatusUpdate对象，再把这个对象传入BaiduMap的animateMapStatus()方法中，就可以将地图移动到指定的经纬度上了:

LatLng ll = new LatLng(39.915,116.404);
MapStatusUpdate update = MapStatusUpdateFactory.newLatLng(ll);
baiduMap.animateMapStatus(update);
让“我”显示在地图上

SDK提供一个MyLocationData.Builder类，这个类用来封装设备当前所在位置的，只需将经纬度传入即可

MyLocationData.Builder locationBuilder = new MyLocationData.Builder();
locationBuilder.latitude(39.915);
locationBuilder.longitude(116.404);

MyLocationData.Builder类还提供了一个build()方法，当我们把要封装的信息都设置完成之后，只需调用它的build()方法，就会生成一个MyLocationData的实例，然后再将这个实例传入BaidMap的setMyLocationData()方法中，就可以让设备当前的位置显示在地图上了:

MyLocationData locationData = locationBuilder.build();
baiduMap.setMyLocationData(locationData);

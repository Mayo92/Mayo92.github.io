深入了解Project

Project详解

module在gradle看来也是project

./gradlew projects

以树的结构进行管理，一般只有两层。每个project必须对应一个build.gradle文件。根project的作用是管理子project。子project用于输出，具体输出什么在它的build.gradle文件配置。

Project核心API讲解

核心API分解

  • project相关API：拥有操作父project以及管理子project的能力
  • task相关API：为当前project提供新增task、使用当前已有task的能力
  • 属性相关API：添加额外属性的能力
  • File相关API：操作当前project下文件的处理
  • gradle生命周期API
  • 其他API

project相关API

每一个groovy脚本都会被编译为script字节码，每一个build.gradle都会被编译为project的字节码，所以相当于在一个project类内部去书写内容

build.gradle中的方法、代码都是在配置阶段执行的

获取所有project

this.getProjects()

def getProjects(){
    println '-------'
    println 'Root project'
    println '-------'
    this.getAllprojects().eachWithIndex {
        Project project,int index ->
            if(index == 0){
                println "Root project ':${project.name}'"
            }else{
                println "+--- project ':${project.name}'"
            }
    }
}

获取所有子project

this.getSubProjects()
def getSubProjects(){
    println '-------'
    println 'project'
    println '-------'
    this.getSubprojects().eachWithIndex {
        Project project,int index ->
            println "+--- project ':${project.name}'"
    }
}

获取父project（在子project中调用）

this.getParentProject()
def getParentProject(){
    def name = this.getParent().name
    println "parent name is ${name}"
}

获取根project

this.getRootPro()
def getRootPro(){
    def name = this.getRootProject().name
    println "root name is ${name}"
}

对子工程进行配置（一般不这么做除非是公共的配置）

project('app') { Project project ->
    println "name is ${project.name}"
    apply plugin: 'com.android.application'
    group 'com.yly'
    version '1.0.1-release'

    dependencies {}

    android {}
}

配置当前工程及其子工程

allprojects {
    group 'com.yly'
}

println project('framework').group

只配置子工程

subprojects { Project project ->
    //库工程上传maven
    if(project.plugins.hasPlugin('com.android.library')){
        apply from: '../publishToMaven.gradle'
    }
}

相对独立的功能可以写到独立的gradle文件中，再由工程引用

属性相关API

自带的属性

  • PATH_SEPARATOR：路径分隔符
  • DEFAULT_BUILD_DIR_NAME：输出文件夹

扩展属性

//定义扩展属性
ext{

}

1.由gradle为每个子工程创建扩展属性，本质是定义多次

subprojects {
    ext {
        compileSdkVersion = 25
    }
}

2.在根工程中定义，子工程获取根工程下的属性，这样只定义了一次

ext {
    compileSdkVersion = 25
}

//app中
compileSdkVersion this.rootProject.compileSdkVersion

根project里的所有属性会被子project继承，所有app中，this.compileSdkVersion也不会报错

3.定义gradle文件

ext {
    //Android 配置
    android = [
            compileSdkVersion: 29
    ]


    signConfigs = [
            'storeFile' : '123.jks'
    ]

    java = [
            'javaVersion' : JavaVersion.VERSION_1_7
    ]

    //依赖 配置
    dependencies = [
            "httpclient"     : 'org.apache.http.legacy'
    ]
    
    //仓库
    maven = [
            "jitpack": "https://jitpack.io"
    ]
}

每一个[]为Map，在里面定义key、value，在根工程下引入

apply from: this.file('common.gradle')

//app下，这里加上rootProject.ext为了避免跟app下的android目录混淆
compileSdkVersion rootProject.ext.android.compileSdkVersion

扩展属性的另一种定义方式

在gradle.properties里定义，只能是key、value形式。

isLoadIjk=true

在setting.gradle中（默认获取的是object类型，所以需要toBoolean）

if(hasProperty('isLoadIjk') ? isLoadIjk.toBoolean() : false){
    include ':ijkplayer-java'
}

注意这种方式，自定义的属性不能与build.gradle文件里已有的方法重复，编译的时候不会报错，使用会找不到

文件属性相关API

作用范围都是同一个根project

文件路径

println "root file path is"+getRootDir().absolutePath
println "project file path is"+getProjectDir().absolutePath
println "build file path is"+getBuildDir().absolutePath

文件定位

def getContent(String path){
    try{
        def file = file(path)
        return file.text
    } catch(GradleException e){
        println 'file not found'
    }
    return null
}

println getContent('common.gradle')

file所传路径为相对应当前project的一个相对路径，files可以传递多个路径得到结果后遍历即可

文件拷贝

copy {
    from file('proguard-rules.pro')
    into getRootProject().getBuildDir()
}

把app下的proguard-rules.pro文件拷贝到根工程的build文件夹下

实际开发中经常需要拷贝生成的APK，下面是拷贝apk文件夹

copy {
    from file('build/outputs/apk/')
    into getRootProject().getBuildDir().path+'/apk/'
}

其他操作

copy {
    from file('proguard-rules.pro')
    into getRootProject().getBuildDir()

    exclude {}//配置不想要的文件
    rename {}//指定文件的重命名
}

文件树遍历

fileTree('build/outputs/apk/') { FileTree fileTree ->
    fileTree.visit { FileTreeElement element ->
        println 'the file name is'+element.file.name
        copy {
            from element.file
            into getRootProject().getBuildDir().path+'/test/'
        }
    }
}

其他API

依赖相关API

buildscript–>ScriptHandler

buildscript { ScriptHandler scriptHandler ->

    //配置工程的仓库地址
    scriptHandler.repositories { RepositoryHandler repositories ->
        repositories.jcenter()
        repositories.mavenCentral()
        repositories.mavenLocal()
        repositories.ivy {}
        repositories.maven {
            name 'personal'
            url 'xxx'
            credentials{
                username = '123'
                password = '123'
            }
        }
    }

    //配置工程的“插件”依赖地址（gradle使用的插件）
    scriptHandler.dependencies {
        classpath 'com.android.tools.build:gradle:3.1.0'
    }
}

project里的dependencies（为应用程序添加第三方依赖）

  1. fileTree、file、files的依赖
  2. 第三方的jar包、aar包的依赖
  3. 源码工程的依赖

两个常见问题

1.依赖冲突解决

exclude两个参数，key和value

2.传递依赖

实际开发禁止传递依赖，因为不确定性

依赖方式比较

compile 第三方库所有的类、资源也会被打包到最终的输出文件中

provided 占位编译，只参与编译不参与打包。还有一种情况，一个库工程需要引入的第三方库已经在被依赖的主工程引入了，库工程可以provided

外部命令执行

exec方法

task(name:'apkcopy') {
    doLast {
        //gradle执行阶段去执行
        def sourcePath = this.buildDir().path + '/outputs/apk'
        def desPath = '/Users/yly/Downloads/'
        def command = "mv -f ${sourcePath} ${desPath}"
    
        exec{
            try{
                executable 'bash' //外部命令
                args '-c',command
                println 'success'
            }catch(GradleException e){
                println 'failed'
            }
        }
    }
}

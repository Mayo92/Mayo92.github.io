Singleton

Singleton优缺点
  • 确保全局至多只有一个对象
  • 用于：构造缓慢的对象，需要统一管理的资源
  • 缺点：很多全局状态，线程安全
Singleton的创建
  • 双重锁模式Double checked locking
  • 作为JAVA类的静态变量
  • 使用框架提供的能力

State模式

继承关系

  • 描述 is-a 关系
  • 不要用继承关系来实现复用
  • 使用设计模式来实现复用

如果Employee升级为Manager？

Decorator模式

变继承关系为组合关系

创建对象

使用new来创建的缺点

  • 编译时必须决定创建哪个类的对象
  • 参数意义不明确

Abstract Factory Pattern

  • task = new LoggingTask(new CodingTask());
  • task = taskFactory.createCodingTask();

Builder Pattern

  • employee = new Employee(oldEmployee.getName(),15000);
  • employee = Employee.fromExisting(oldEmployee).withSalary(15000).build();
  • 不可变对象往往配合Builder使用

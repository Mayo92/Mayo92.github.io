索引的实现

  • B树
  • B+树

二叉树优化比较次数，B/B+树优化磁盘读写次数

B树

B+树

非叶子节点都是用来帮助索引的节点，数据在叶子节点

m阶B/B+树

  • 每个非叶子节点（除根外）至多有m个儿子，至少有[m/2]个儿子
  • 根节点（如果不是叶子）至少有两个儿子
  • 所有叶子节点在同一层

数据库索引的分类

  • Clustered Index，每个表至多一个
  • Non-clustered Index

clustered快，但是显然只能建一个，我们为数据库建立的主键，后面一般都有clustered index。non-clustered虽然慢一些，但能建很多个，它比起没有索引还是快很多，我们可以针对我们经常需要搜索的字段去建non-clustered index，来加快我们的搜索

数据库连接池

连接池的配置（Tomcat JDBC Connection Pool）

  • maxActive（默认100）
  • maxIdle（默认100）
  • minIdle（默认10）
  • initialSize（默认10）
  • maxWait（默认30000，30秒）

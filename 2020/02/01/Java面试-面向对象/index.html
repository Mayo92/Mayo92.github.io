Java面试-面向对象

2020-02-01
#Java面试

面向对象思想

  • 从用户（终端用户、使用你代码的用户）角度思考问题
  • 摒弃完全基于逻辑的思维

类与对象

  • 类的成员变量 -> 对象状态
  • 类的函数变量 -> 对象行为
  • 类的静态变量
  • 类的静态函数

物理结构：

通过this与对象相关联

静态变量、静态函数

  • 没有this引用，静态变量全局唯一一份
  • 普通函数可以引用静态变量、函数
  • 对象上引用静态变量、函数？ 编译器警告
  • 静态函数引用普通成员变量、函数？ 编译错误（没有this，不知道是哪个对象）

类的特殊函数

构造函数、equals、hashCode、toString

object的equals方法

如果equals那么hashCode一定相等

java1.7之后可以简化写法

接口

为什么要有接口

  • 从用户（使用实现的代码）角度看问题

与类相比

  • 由编译器强制的一个模块间协作的合约（contract）
  • 没有成员变量
  • 成员函数只有声明不能实现

接口与抽象类

从实现角度

  • 抽象类可以有成员变量
  • 抽象类可以有部分实现
  • 抽象类不可以多重继承，接口可以

接口可以有常量，JDK1.8可以有带方法体的静态方法

继承

  • is-a 的关系
  • 子类增加或修改基类（增加成员变量、函数）
  • java中所有的函数都相当于C++虚函数

封装

尽量只使用private和public

派生类中修改封装可见性

private -> public 可以

public -> private 不可以

不建议，最好遵从。可以增加可见性（子类只能增加或者修改基类）

不可变性

不可变对象Immutable Objects

  • 可以引用传递，可以缓存
  • 线程安全

final关键字

  • 类声明 -> 类不可以被继承
  • 函数声明 -> 函数不可以在派生类中重写
  • 变量声明 -> 变量不可以指向其他对象（Object对象的内容可以修改）
  • static final变量 -> 用于定义常量，名称一般大写

实现不可变性

  • final关键字无法保证不可变性
  • 从接口定义，类的实现上保证不可变性
  • Collections.unmodifiableXXX

需要拷贝一份再调用Collections.unmodifiableXXX方法。因为unmodifiableList是一个view，我们看到的和外面传进来的reporters其实是同一个list。只是因为包了一层unmodifiableList，在Manager类里面改不了了。但是在外面还是可以改的，并且会体现到Manager里面。而我们使用new ArrayList拷贝了一份之后，就杜绝了外面有人可以改变它。

  • ArrayLis，ArrayLis
